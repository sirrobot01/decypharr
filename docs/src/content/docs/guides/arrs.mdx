Decypharr provides both QBitTorrent and Sabnzbd-compatible APIs for Sonarr, Radarr, and other *Arr applications.

## For Torrents/Debrid: QBitTorrent Client

In Sonarr/Radarr:

1. Go to **Settings** → **Download Clients**
2. Click **+** → **qBittorrent**

### QBitTorrent Settings

| Field | Value |
|-------|-------|
| **Name** | Decypharr (Torrents) |
| **Host** | `decypharr` (or IP) |
| **Port** | `8282` |
| **Username** | Arr's host(`http://sonarr:8989`) |
| **Password** | Arr's token(gotten from Arr's General -> Token) |
| **Category** | `sonarr` / `radarr` |
| **Priority** | `0` (Normal) |

### Advanced

```
Remove Completed: Yes
Remove Failed: No
```

Click **Test** → **Save**

### Can I use both Debrid and Usenet in same Arr?

Yes! See [Using Both Torrents and Usenet](#using-both-torrents-and-usenet) above.

Add both clients with different priorities to prioritize one source over the other.you prefer one over the other. Arrs will use the higher-priority client first.

## Using Both Torrents and Usenet

You can add **both** download clients to the same Arr:

- **Decypharr (Torrents)** - QBitTorrent client
- **Decypharr (Usenet)** - Sabnzbd client

Set different priorities if you prefer one over the other. Arrs will use the higher-priority client first.

## Path Mapping

If your Arr and Decypharr are on different systems, configure path mappings.

### Example Setup

- **Decypharr** mount: `/mnt/decypharr`
- **Sonarr** sees: `/media/tv`

**Path Mapping:**
```
Remote Path: /mnt/decypharr
Local Path: /media/tv
```

## Download Actions

Decypharr supports multiple download actions configured per-Arr or globally:

| Action | Description |
|--------|-------------|
| `symlink` | Create symlink to mounted file (default) |
| `download` | Download file to local storage |
| `strm` | Create STRM file pointing to WebDAV URL |
| `none` | No action (Arr manages files) |

Set in `config.json`:

```json
{
  "arrs": [
    {
      "name": "Sonarr",
      "host": "http://sonarr:8989",
      "token": "ARR_API_KEY",
      "download_action": "symlink"
    }
  ]
}
```

## Queue Cleanup

Sonarr/Radarr queues often get stuck on releases that can't import automatically — failed
grabs, title mismatches, empty files, and so on. Decypharr watches every connected Arr's
**Activity → Queue** and resolves these for you, so they don't pile up and block your library.

You configure this **once, globally** (it is not per-Arr) under
**Settings → Arrs → Queue Cleanup Actions** in the Web UI. Each kind of problem maps to one of
four actions:

| Action                   | What it does                                                            |
|--------------------------|------------------------------------------------------------------------|
| **Ignore**               | Leave it in the queue (default for ambiguous cases)                     |
| **Force import**         | Try a manual import of the downloaded files                            |
| **Blacklist only**       | Blocklist + remove the release, but **don't** grab a replacement       |
| **Blacklist + research** | Blocklist + remove the release **and** search for a new one            |

### Known issues

The UI ships with a catalog of the common Sonarr/Radarr warnings, each with a sensible
default you can override from a dropdown:

| Issue                                            | Default action       |
|--------------------------------------------------|----------------------|
| Failed download                                  | Blacklist + research |
| Title mismatch (automatic import not possible)   | Force import         |
| Release matched to series/movie by ID            | Force import         |
| Unable to parse download                         | Blacklist + research |
| No files eligible for import                     | Blacklist + research |
| Episodes missing / not imported from release     | Blacklist + research |
| Downloaded file is empty                         | Blacklist + research |
| Invalid local path (remote path mapping)         | Ignore               |
| Not grabbed by the Arr / no category             | Ignore               |

### Custom rules

Need to handle something not in the list? Click **Add custom rule** and enter any text that
appears in the queue item's status message (case-insensitive) plus the action to take. For
example, match `stalled with no connections` and set **Blacklist + research**.

Known issues are always evaluated before custom rules, and the **first match wins**. Anything
that doesn't match any rule is left in the queue. Changes take effect on the next cleanup
cycle — no restart needed.

:::note
The raw `queue_cleanup` config shape and the full list of catalog rule IDs are documented in
[Configuration → Queue Cleanup](../configuration/#queue-cleanup).
:::

## Auto-Detection

Decypharr can auto-detect Arrs that connect to it:

```json
{
  "arrs": [
    {
      "name": "Sonarr (Auto)",
      "host": "http://sonarr:8989",
      "token": "ARR_API_KEY",
      "source": "auto"
    }
  ]
}
```

Check **Settings** → **Arrs** in the Web UI to see detected instances.

## Troubleshooting

### Test Connection Fails

1. Verify Decypharr is accessible: `curl http://decypharr:8282/version`
2. Check username/password match config
3. Ensure no firewall blocking port 8282

### Downloads Not Importing

**Check path mapping** - Arr must see files at same path structure.

**Example (Docker):**

```yaml
# docker-compose.yml
services:
  decypharr:
    volumes:
      - /mnt/storage:/mnt/storage  # Same path!
  
  sonarr:
    volumes:
      - /mnt/storage:/mnt/storage  # Same path!
```

### Uncached Torrents

Enable `download_uncached` per-Arr:

```json
{
  "arrs": [
    {
      "name": "Sonarr",
      "download_uncached": true
    }
  ]
}
```

:::caution
Uncached downloads use Debrid provider's cache slots and may take time to download.
:::
